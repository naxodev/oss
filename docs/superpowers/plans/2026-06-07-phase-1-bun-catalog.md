# Phase 1 — Package manager (pnpm → bun) + catalog — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace pnpm with bun as the workspace package manager and centralize dependency versions in a root bun `catalog`, with no behavioral change to the published plugins or test suites yet.

**Architecture:** Add bun workspaces (`workspaces: ["packages/*"]`) so bun sees the member packages, generate `bun.lock`, and move every root dependency version into a root `catalog` referenced via `catalog:`. Swap repo-level `pnpm exec nx` tooling to `bunx nx` in configs, CI, husky, and docs. Keep `pnpm/action-setup` in CI **only** as the binary the e2e harness shells out to for its throwaway test workspace (that harness is migrated in Phase 3). The two publishable plugins' runtime `dependencies` stay as explicit version ranges this phase — their `catalog:` conversion lands in Phase 4 together with the publish-time substitution script, so every committed state remains publish-valid.

**Tech Stack:** bun 1.3.x, Nx 22.7.5, TypeScript, GitHub Actions, husky/commitlint.

**Spec:** `docs/superpowers/specs/2026-06-07-bun-ts-project-linking-migration-design.md` (this is Phase 1 of 4; Phases 2–4 are planned after each verification gate).

### Task 2: Introduce the root catalog

**Files:**

- Modify: `package.json` (add `catalog`, convert root `dependencies`/`devDependencies` to `catalog:`)
- Modify: `bun.lock` (regenerated)

- [ ] **Step 1: Transform root deps into a catalog (deterministic script)**

Run this exact node script from the repo root. It moves every current root `dependencies` + `devDependencies` version into a `catalog` block (single source of truth) and rewrites each entry to `catalog:`. It preserves key order of dep blocks and leaves `nx` config / `workspaces` untouched.

```bash
node -e '
const fs = require("fs");
const p = JSON.parse(fs.readFileSync("package.json", "utf8"));
const catalog = { ...(p.catalog || {}) };
for (const field of ["dependencies", "devDependencies"]) {
  const block = p[field];
  if (!block) continue;
  for (const [name, spec] of Object.entries(block)) {
    if (typeof spec !== "string" || spec.startsWith("catalog:") || spec.startsWith("workspace:")) continue;
    if (catalog[name] && catalog[name] !== spec) {
      throw new Error(`catalog conflict for ${name}: ${catalog[name]} vs ${spec}`);
    }
    catalog[name] = spec;
    block[name] = "catalog:";
  }
}
// Emit catalog with sorted keys for a stable, reviewable diff.
const sorted = {};
for (const k of Object.keys(catalog).sort()) sorted[k] = catalog[k];
// Insert catalog after "private" (or at top of object tail) by rebuilding.
const { name, version, license, scripts, private: priv, workspaces, ...rest } = p;
const out = { name, version, license, scripts, private: priv, workspaces, catalog: sorted, ...rest };
fs.writeFileSync("package.json", JSON.stringify(out, null, 2) + "\n");
console.log(`catalog now has ${Object.keys(sorted).length} entries`);
'
```

Expected: prints `catalog now has <N> entries` (N ≈ 75), exits 0.

- [ ] **Step 2: Re-install and verify resolution is unchanged**
